## 판단 기준 (Few-Shot 예시 — 실제 장애 사례 기반)

### 예시 1: 위험 — 심층 분석 필요
함수: c400_get_svc_list
탐지 패턴:
- UNINIT_VAR: `long curcnt, ix;` (초기화 없음)
- BUFFER_INDEX: `svclist[curcnt]` (미초기화 변수로 배열/루프 접근)
판정: **위험** — 루프 제어 변수(curcnt, ix)가 초기화되지 않아 쓰레기 값으로 루프가 미실행되거나 비정상 반복됩니다. 올바른 코드는 `long curcnt = 0, ix = 0;`입니다. (실제 장애: 루프 미실행으로 데이터 누락)

### 예시 2: 위험 — 심층 분석 필요
함수: c100_abn_sale_proc
탐지 패턴:
- BOUNDED_FUNC: `memset(&zord_abn_sale_spc_u0010_in, 0x00, sizeof(zord_abn_sale_spc_s0009_in_t));`
판정: **위험** — memset 대상 변수명(u0010)과 sizeof 타입명(s0009)이 불일치합니다. 구조체 크기가 다르면 일부만 초기화되어 이전 트랜잭션 데이터가 잔류합니다. 변수명과 sizeof 타입의 숫자 부분이 다르면 반드시 의심하세요. (실제 장애: 이전 데이터 잔류로 오처리)

### 예시 3: 위험 — 심층 분석 필요
함수: c200_auto_chrg_proc
탐지 패턴:
- BOUNDED_FUNC: `PFM_ERRS(" [%s] 자동선결제 인증요청이력이 존재하지 않습니다.", ctx->svc_mgmt_num);`
판정: **위험** — svc_mgmt_num은 long형 변수인데 포맷 지정자가 %s(문자열)입니다. long 값을 문자열 포인터로 해석하면 잘못된 메모리 접근으로 Core Dump가 발생합니다. 올바른 포맷은 `%ld`입니다. PFM_ERRS, snprintf 등에서 포맷 지정자(%ld, %d, %s)와 인자 타입이 반드시 일치하는지 확인하세요. (실제 장애: Core Dump 발생)

### 예시 4: 안전 — 생략
함수: a000_init_proc
탐지 패턴:
- UNINIT_VAR: `long rc;` (그러나 직후 `rc = pfm_init(ctx);`로 대입)
판정: **안전** — 선언 직후 함수 호출 결과로 초기화되므로 실제 위험 없습니다. ProFrame 코드에서 `long rc;` 후 즉시 대입하는 것은 표준 패턴입니다.

### 예시 5: 안전 — 생략
함수: z999_err_exit_proc
탐지 패턴:
- BOUNDED_FUNC: `memset(&ordsb0510010t04_ctx, 0x00, sizeof(ordsb0510010t04_ctx_t));`
판정: **안전** — memset 대상 변수(ordsb0510010t04_ctx)와 sizeof 타입(ordsb0510010t04_ctx_t)의 숫자 부분이 동일하므로 크기 일치가 보장됩니다. 예시 2와 비교하여 숫자가 일치하는지 여부가 핵심 판별 기준입니다.

## 주의사항
- 실제 위험이 있는 함수만 선별하세요. 패턴이 있어도 안전하면 제외합니다.
- UNINIT_VAR는 선언 직후 대입(`= ...`)이 있으면 안전합니다.
- BOUNDED_FUNC는 sizeof 대상과 버퍼가 일치하면 안전합니다.
- 의심스러우면 포함하세요 — Pass 2에서 LLM이 정밀 분석합니다.

## ⛔ 선별 제외 대상 (반드시 무시할 오류 유형)
분석 환경은 가짜 헤더(stub)를 사용하고 있으므로, 다음과 같은 원인으로 발생한 패턴은 **절대 실제 위험으로 간주하여 선별하지 마세요.**
- **미정의 타입/식별자 오류**: 프레임워크 공통 헤더에 정의된 `typedef`, `struct`, `#define` 매크로 부재로 인한 파싱 실패 오류
- **암시적 함수 선언 오류**: 외부 함수 프로토타입 누락 (예: `PFM_DBG` 등 프레임워크 함수가 없다는 내용)
- **헤더 미발견 오류**: 분석 환경에 프레임워크 관련 헤더(`pfmcom.h` 등)가 없어서 생기는 오류
- **축소 변환 오류 (int → char 등)**: 암시적 선언으로 int 반환값을 가지게 된 외부 함수 호출 결과를 타입 변수에 대입할 때 발생하는 가짜 정적 오류
- **strncpy 널 종료 미보장 (LEN_* 사용)**: 프로프레임(ProFrame) 환경에서는 변수가 자동 선언 시 `+ 1`이 할당되므로 `strncpy`에 `LEN_...` 상수를 쓸 때 버퍼 널-종료가 안전하게 보장됩니다. 의심 대상으로 선별하지 마세요.

당신은 소스코드 분석 결과의 배포 위험도를 한국어로 요약하는 전문가입니다.

## 입력 데이터
{analysis_results}

입력에는 `by_severity` (critical/high/medium 이슈 개수)와 `deployment_risk` (CRITICAL/HIGH/MEDIUM/LOW 판정)만 포함됩니다.

## 작업
위 통계와 배포 판정을 바탕으로 **한국어 배포 위험 설명** 1개를 작성합니다.

작성 기준:
- **2~3 문장** 이내로 간결하게
- 실제 이슈 개수와 배포 판정을 명시
- 과장 없이 사실 기반으로. 이슈 0건이면 "문제 없음" 수준으로 처리
- bullet point나 리스트 금지 — 산문(文)으로만

## 출력 형식 (엄격)

반드시 아래 JSON 구조만 반환. 다른 필드는 추가하지 말 것:

{{
  "summary": {{
    "risk_assessment": {{
      "risk_description": "한국어 2~3 문장 요약"
    }}
  }}
}}

## 예시

입력: `{{"by_severity": {{"critical": 2, "high": 1, "medium": 0}}, "deployment_risk": "CRITICAL"}}`

출력:
{{
  "summary": {{
    "risk_assessment": {{
      "risk_description": "Critical 이슈 2건과 High 이슈 1건이 발견되어 배포가 차단됩니다. Critical 항목을 우선 수정 후 재검증이 필요합니다."
    }}
  }}
}}

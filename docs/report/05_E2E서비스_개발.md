## 1. E2E 파이프라인 흐름

### 전체 흐름도

```

   │                                  │
   │  mider.exe --files *.c *.pc      │
   │ ────────────────────────────────→ │
   │                                  │
   │                           ┌──────▼──────┐
   │                           │  main.py     │
   │                           │  .env 로드    │
   │                           │  API 키 검증   │
   │                           │  모델 결정     │
   │                           └──────┬──────┘

   │                           ┌──────▼──────────────┐

   │                                  │

```

## 2. CLI 인터페이스

### 2.1 실행 방법

```bash
# 폐쇄망 (Windows)
mider.exe --files src/calc.c src/order.pc

# 폐쇄망 (Linux)
./mider --files src/calc.c src/order.pc

# 개발 환경
python main.py --files src/calc.c src/order.pc

# 전체 옵션
python main.py --files "src/**/*.c" "src/**/*.pc" \
               --explain-plan explain_output.txt \
               --output ./reports/0315 \
               --model gpt-5 \
               --verbose
```

### 2.2 CLI 옵션

| 옵션 | 단축 | 기본값 | 설명 |
|------|------|--------|------|
| `--files` | `-f` | **(필수)** | 분석할 파일 경로 (1개 이상, glob 지원) |
| `--output` | `-o` | `./output` | 결과 출력 디렉토리 |
| `--model` | `-m` | settings.yaml | LLM 모델명 (CLI > 환경변수 > settings.yaml) |
| `--explain-plan` | `-e` | `None` | Explain Plan 결과 파일 (SQL 분석 시) |
| `--verbose` | `-v` | `false` | 상세 로그 + 추론 과정 출력 (ReasoningLogger) |
| `--version` | | | 버전 정보 출력 (v0.1.0) |

### 2.3 환경 변수

| 변수명 | 기본값 | 설명 |
|--------|--------|------|
| `AZURE_OPENAI_API_KEY` | - | Azure OpenAI API 키 |
| `AZURE_OPENAI_ENDPOINT` | - | Azure OpenAI 엔드포인트 |
| `OPENAI_API_KEY` | - | OpenAI API 키 (Azure 미설정 시) |
| `MIDER_API_KEY` | - | OPENAI_API_KEY로 브리징 |
| `MIDER_API_BASE` | - | API Base URL (폐쇄망 프록시) |
| `MIDER_MODEL` | - | 기본 LLM 모델 오버라이드 |
| `MIDER_LOG_LEVEL` | `INFO` | 로그 레벨 |
| `MIDER_EXCLUDE_FUNCTIONS` | - | 분석 제외 함수명 (대형 파일 임시 우회) |

API 키 우선순위: Azure (`AZURE_OPENAI_API_KEY` + `AZURE_OPENAI_ENDPOINT`) > `MIDER_API_KEY` > `OPENAI_API_KEY`

### 3.2 JSON 출력 파일 (4개)

| 파일 | 스키마 | 내용 |
|------|--------|------|
| `issue-list.json` | IssueList | 전체 이슈 목록 (severity순 정렬, Before/After 코드 포함) |
| `checklist.json` | Checklist | Critical/High 검증 체크리스트 (검증 명령어 + 기대 결과) |
| `summary.json` | Summary | 분석 메타데이터 + 이슈 통계 + 배포 판정 (risk_description) |
| `deployment-checklist.json` | DeploymentChecklist | 5개 섹션 배포 절차 (화면/TP/모듈/배치/DBIO) |

## 4. 종료 코드

| 코드 | 의미 | 배포 |
|------|------|------|
| `0` | 정상 완료, Critical 없음 | 가능 |
| `1` | 정상 완료, Critical 있음 | 불가 |
| `2` | 실행 오류 (파일 없음, 권한 없음, 확장자 미지원) | - |
| `3` | LLM API 오류 (API 키 미설정, 네트워크 오류, 타임아웃) | - |

## 5. 배포 판정 로직

```
분석 완료
  │
  ▼
Critical > 0? ──Yes──→ CRITICAL (배포 차단, 즉시 수정 필요)
  │ No
  ▼
High >= 3?   ──Yes──→ HIGH (배포 차단, 수정 권고)
  │ No
  ▼
High >= 1?   ──Yes──→ MEDIUM (배포 가능, 수정 권고)
  │ No
  ▼
              LOW (배포 가능)
```

## 6. 폐쇄망 배포

### 6.1 실행파일 패키징

* Python 앱 → 단일 실행파일(.exe) 형태로 패키징
* pip install, npm install 불필요 (제로 설정)
* 정적 분석 바이너리(ESLint/clang-tidy/proc) 포함

### 6.2 지원 환경

| 환경 | 실행 방법 |
|------|----------|
| Windows (폐쇄망) | `mider.exe --files ...` |
| Linux (폐쇄망) | `./mider --files ...` |
| 개발 환경 | `python main.py --files ...` |

### 6.3 LLM 연동

* Azure OpenAI: 폐쇄망 내 Azure 서버 → `AZURE_OPENAI_ENDPOINT` 설정
* OpenAI 프록시: 폐쇄망 프록시 서버 → `MIDER_API_BASE` 설정
* 모델: gpt-5 (Primary), gpt-5-mini (Fallback/경량), gpt-4.1-mini (Reporter)

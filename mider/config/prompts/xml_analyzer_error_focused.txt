## 분석 절차 (Chain-of-Thought)

### Step 1: 파서 오류 분석
1. XML 파싱 오류: 잘못된 태그 구조, 네임스페이스 오류 등
2. 중복 ID: 같은 ID가 여러 컴포넌트에 사용됨 → 런타임 오작동
3. JS 핸들러 누락: ev:onclick 등으로 바인딩했으나 JS에 함수 미정의

### Step 2: 심층 구조 분석

**데이터 바인딩 (data_integrity)**
- dataList에 정의된 컬럼이 실제 화면 컴포넌트와 바인딩되지 않음
- dataType이 화면 입력 유형과 불일치 (예: number 타입에 text 입력)

**이벤트 처리 (error_handling)**
- 이벤트 핸들러에서 에러 처리 누락
- 복수 이벤트가 동일 함수를 호출하여 의도치 않은 중복 실행

**보안 (security)**
- 사용자 입력 컴포넌트에 유효성 검사(validation) 설정 누락
- 숨김(hidden) 필드에 민감 정보 직접 바인딩

**성능 (performance)**
- 불필요하게 많은 dataList 정의 (실제 사용하지 않는 데이터 리스트)
- 과도한 이벤트 바인딩 (같은 요소에 중복 이벤트)

## 이슈 예시
{{
  "issue_id": "XML-001",
  "category": "data_integrity",
  "severity": "high",
  "title": "중복 컴포넌트 ID로 인한 런타임 오작동 위험",
  "description": "ID 'btn_search'가 2개 컴포넌트에서 사용되고 있습니다. WebSquare는 getElementById로 컴포넌트를 찾으므로 한쪽이 무시되어 이벤트가 동작하지 않을 수 있습니다.",
  "location": {{
    "file": "{file_path}",
    "line_start": 0,
    "line_end": 0
  }},
  "fix": {{
    "before": "id=\"btn_search\" (2개소)",
    "after": "id=\"btn_search\", id=\"btn_search2\"",
    "description": "컴포넌트 ID를 고유하게 변경합니다."
  }},
  "source": "static_analysis"
}}

      "description": "한국어 상세 설명",

        "after": "수정 후 코드 (1-3줄)",

      "static_rule": null

  ]
}}

## 주의사항
- 이슈 ID는 XML-001부터 순차적으로 부여합니다.
- title과 description은 반드시 한국어로 작성합니다.
- fix의 before/after는 1-3줄만 포함합니다.
- 발견된 이슈가 없으면 빈 배열 `"issues": []`로 응답합니다.
- 중복 ID, 핸들러 누락은 source: "static_analysis"로 표시합니다.
- **[중요] 서로 다른 dataList 간 동명 컬럼은 절대 이슈로 보고하지 마세요.** 각 dataList는 완전히 독립된 스코프입니다. 예: DS_A의 "col_x"와 DS_B의 "col_x"는 전혀 다른 컬럼이며, 이름이 같아도 중복/불일치/충돌이 아닙니다. dataList 간 컬럼 비교 자체를 하지 마세요.
- severity 기준:
  - critical: 화면 렌더링 실패 또는 데이터 손실
  - high: 특정 조건에서 기능 오작동
  - medium: 구조적 품질 저하, 잠재적 버그
  - low: 스타일, 가독성 개선 사항

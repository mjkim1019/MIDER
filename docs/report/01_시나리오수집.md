# 1단계: 시나리오 수집

## 핵심 사용자 시나리오

### **시나리오 1:** 배포 전 단일 파일 긴급 검증

* **ID**: SC-001

**상황**: 폐쇄망 환경에서 1년차 개발자가 급여 계산 배치(Pro*C) 파일을 수정한 후, 배포 전 코드 검증이 필요한 상황. 시니어 개발자는 장기 휴가중

**목표**: 수정한 파일의 Critical/High 이슈를 당일 내 파악하여 배포 가능 여부를 즉시 판단

**사전 조건**:

* CLI 도구가 폐쇄망 PC에 설치됨 (실행파일 형태, pip install 불필요)
* Azure OpenAI API 서버 정상 동작 (환경변수: AZURE_OPENAI_API_KEY, AZURE_OPENAI_ENDPOINT)
* 분석 대상 파일이 로컬에 존재

**상세 흐름**

| 단계 | 사용자 행동 | 시스템 동작 | 비고 |
| -- | -------- | -------- | --- |
| 1 | Mider 실행파일(CLI) 실행 및 분석 대상 파일 지정 | | |
| 2 | 명령어 입력: `mider.exe --files batch/salary_calc.pc` | - OrchestratorAgent 시작<br>- TaskClassifierAgent가 파일 언어 감지 (Pro*C)<br>- ExecutionPlan 생성 | Phase 0 |
| 3 | 대기 (진행률 확인) | - ContextCollectorAgent가 EXEC SQL 블록, #include 파일목록 추출<br>- FileContext 생성 | Phase 1 |
| 4 | 대기 (진행률 확인) | - ProCAnalyzerAgent 실행<br>  1. Oracle proc 실행 → SQLCA 체크 누락 발견<br>  2. ProCHeuristicScanner → regex 패턴 탐지<br>  3. LLM 심층 분석 (gpt-5, Error-Focused 경로) | Phase 2 |
| 5 | 대기 (진행률 확인) | - ReporterAgent가 4개 리포트 생성<br>  • issue-list.json (Before/After 코드)<br>  • checklist.json (검증 명령어)<br>  • summary.json (통계, 배포 판정)<br>  • deployment-checklist.json (배포 체크리스트) | Phase 3 |
| 6 | 터미널 출력 확인 | - Critical 1건, High 1건 표시<br>- "배포 판정: 불가 (Critical 1건)" 메시지<br>- 종료 코드: 1 (Critical 있음) | 최종 출력 |
| 7 | 로컬에 저장된 issue-list.json 열람 및 수정 작업 시작 | Critical: SQLCA 체크 누락 (line 89)<br>- Before/After 코드 확인 | |

* 4개 JSON 리포트 정상 생성
* Critical 존재 시 종료 코드 1 반환

### **시나리오 2:** 배포 전 여러 파일 동시 검증

* ID: SC-002

* 상황: 폐쇄망 환경에서 시니어 개발자가 고객 보유 상품의 실시간 가격 조회 기능 개선 작업 완료 후, UI(JS) → TP(Service C) → DBIO(SQL) 3개 파일을 동시에 배포 전 검증

* 목표: 3계층 아키텍처 전체의 Critical 이슈를 파악하고 수정 우선순위 결정

* 사전 조건:

  * CLI 도구 설치 완료, LLM API 서버 정상 동작
  * 분석 대상 3개 파일 준비: ui/product_price.js (화면), service/get_price_tp.c (TP), dbio/select_price.sql (DB 조회)

**상세 흐름**

| 단계 | 사용자 행동 | 시스템 동작 | 비고 |
| -- | -------- | -------- | --- |
| 1 | 3계층 파일 지정: `mider.exe --files ui/product_price.js service/get_price_tp.c dbio/select_price.sql` | - 3개 파일 언어 감지 (JavaScript, C, SQL)<br>- 의존성 분석<br>- ExecutionPlan 생성 (우선순위 자동 부여) | Phase 0 |
| 2 | 대기 | - ContextCollectorAgent<br>- 3개 파일의 import/함수 호출 추출<br>- FileContext 생성 | Phase 1 |
| 3 | 대기 (진행률 확인) | **[1/3] dbio/select_price.sql**<br>- SQLAnalyzerAgent: sqlparse 문법 검증 + 정적 패턴 분석 + LLM<br>**[2/3] service/get_price_tp.c**<br>- CAnalyzerAgent: clang-tidy/Heuristic + LLM<br>- Critical 1건, High 1건 발견<br>**[3/3] ui/product_price.js**<br>- JavaScriptAnalyzerAgent: ESLint + LLM<br>- High 1건 발견 (이벤트 리스너 cleanup 누락) | Phase 2 |
| 4 | 대기 (진행률 확인) | - ReporterAgent: 전체 이슈 통합<br>- 심각도별 분류 + 정렬<br>- 4개 JSON 리포트 생성<br>- 배포 체크리스트 (화면/TP/모듈/DBIO 섹션 자동 분류) | Phase 3 |
| 5 | 출력 확인 | - Critical 2건 (SQL, C)<br>- High 2건 (C, JS)<br>- Medium 1건 (SQL)<br>- "배포 판정: 불가 (Critical 2건)" 메시지 | |
| 6 | 로컬에 저장된 리포트 파일 확인 | - issue-list.json: 5개 이슈 (심각도순 정렬)<br>- checklist.json: Critical/High 검증 항목<br>- summary.json: 파일별/심각도별 통계<br>- deployment-checklist.json: 5개 섹션별 배포 항목 | |

**입력 예시**

```bash
# 기본 사용
mider.exe --files ui/product_price.js service/get_price_tp.c dbio/select_price.sql

# glob 패턴
mider.exe --files "src/**/*.c" "src/**/*.pc"

# SQL + Explain Plan
mider.exe --files dbio/select_price.sql --explain-plan explain_output.txt

# 출력 디렉토리 지정
```

### 지원 언어 및 분석 방식

| 언어 | 확장자 | 정적 분석 도구 | Analyzer Agent |
| ---- | ----- | ----------- | -------------- |
| JavaScript | .js | ESLint | JavaScriptAnalyzerAgent |
| C | .c, .h | clang-tidy + CHeuristicScanner | CAnalyzerAgent |
| Pro*C | .pc | Oracle proc + ProCHeuristicScanner | ProCAnalyzerAgent |
| SQL | .sql | sqlparse + ExplainPlanParser | SQLAnalyzerAgent |
| XML (WebSquare) | .xml | XMLParser (ElementTree) | XMLAnalyzerAgent |

### 종료 코드 체계

| 코드 | 의미 | 동작 |
|------|------|------|
| `0` | 정상 완료, Critical 없음 | 배포 가능 |
| `1` | 정상 완료, Critical 있음 | 배포 불가 |
| `2` | 실행 오류 (파일 없음, 권한 없음) | 분석 중단 |
| `3` | LLM API 오류 | 분석 중단 |

### 배포 판정 기준

| 조건 | 판정 | 배포 |
|------|------|------|
| Critical > 0 | CRITICAL | 차단 |
| Critical = 0, High >= 3 | HIGH | 차단 |
| Critical = 0, High >= 1 (< 3) | MEDIUM | 가능 (수정 권고) |
| Critical = 0, High = 0 | LOW | 가능 |

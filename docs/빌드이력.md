## 재빌드 체크리스트

> **`.env`를 수정하면 반드시 재빌드해야 한다** (onefile 모드에서 exe 내부에 번들됨)

### 사전 조건

```bash
# 1. Python 의존성 설치 확인
pip install -r requirements.txt
```

필수 패키지: `openai`, `httpx`, `pydantic`, `rich`, `sqlparse`, `selenium`, `python-dotenv`, `pyyaml`

```bash
# 2. 바이너리 파일 확인
#    아래 파일들이 mider/resources/binaries/ 에 존재해야 함
#    없으면 원본 mider 프로젝트에서 복사:
#      D:\COE\mider\mider\resources\binaries\ → mider_prd2\mider\resources\binaries\
```

| 파일 | 크기 | 용도 | 필수 |
|------|------|------|------|
| `node.exe` | 82MB | ESLint 실행용 포터블 Node.js (v22.17.1) | O |
| `clang-tidy.exe` | 64MB | C 정적분석 도구 (LLVM 18.x) | O |
| `libclang.dll` | 96MB | clang-tidy 런타임 라이브러리 | O |
| `node_modules/` | 14MB | ESLint 8.57.1 + npm 의존성 | O |

```bash
# 3. .env 파일 확인 (없으면 생성)
cp .env.example .env
# 반드시 실제 API 키/엔드포인트를 설정한 후 빌드할 것
```

### 빌드 실행

```bash
cd D:\COE\mider_prd2
python -m PyInstaller --clean --noconfirm mider.spec
```

### 빌드 검증

```bash
# 버전 확인
dist\mider.exe --version

# 도구 상태 점검 (node, ESLint, clang-tidy, config, selenium, LLM)
dist\mider.exe --check
```

## 빌드 시 주의사항

| 항목 | 설명 |
|------|------|
| `.env` 변경 시 | **반드시 재빌드 필요** — exe 내부에 번들되므로 외부에서 교체 불가 |
| 바이너리 누락 시 | `mider/resources/binaries/`에 node.exe, clang-tidy.exe 등 없으면 ESLint/clang-tidy 실행 불가 |
| node_modules 누락 시 | ESLint 실행 불가 — `mider.spec`이 디렉토리를 재귀적으로 번들함 |
| selenium 오류 시 | `mider.spec`의 `hiddenimports`에 selenium 서브모듈 포함 필수 |
| 한글 깨짐 | Windows cp949 인코딩 이슈 — 실행에 영향 없음 |

## 빌드 이력

### 2026-04-13 (4차 빌드 — 최종)

**변경:**
- `mider/healthcheck.py` 신규 생성 — `--check` 명령으로 6개 항목 검증
  - node, ESLint, clang-tidy, config, selenium, LLM API
- `mider/main.py` — `--check` CLI 인자 + 분기 추가
- `mider.spec` — hiddenimports에 `mider.healthcheck`, selenium 서브모듈 추가

**selenium hidden imports 이슈:**
SSO 로그인 시 `No module named 'selenium.webdriver.chrome.options'` 오류 발생.
`mider.spec`의 hiddenimports에 아래 항목 추가로 해결:
```
selenium, selenium.webdriver, selenium.webdriver.chrome,
selenium.webdriver.chrome.options, selenium.webdriver.chrome.service,
selenium.webdriver.common, selenium.webdriver.common.by,
selenium.webdriver.support, selenium.webdriver.support.ui,
selenium.webdriver.support.expected_conditions
```

### 2026-04-13 (1차 빌드 — 초기)

**수행 단계:**
1. 의존성 설치 (`pip install -r requirements.txt`)
2. 바이너리 복사 (mider → mider_prd2: node.exe, clang-tidy.exe, libclang.dll, node_modules/)
3. `.env` 파일 생성 (`.env.example` 복사)
4. `mider.spec` 수정 — node_modules 디렉토리 재귀 포함 로직 추가
5. PyInstaller 빌드 실행
6. `mider.exe --version` → `mider 1.0.0` 검증 완료

# 이슈 #007: C 분석 이슈 중복 — 함수별 LLM이 동일 패턴 독립 보고

## 발견일
2026-03-17

## 발견 경위
2932줄 C 파일(ordsb0100010t01.c) 분석 시 2-Pass 경로에서 5개 함수를 개별 LLM 호출.
각 함수가 독립적으로 strncpy 널 종료, 스레드 안전성 등을 보고하여 60건 → 29건(프롬프트 개선 후)의 이슈 중 14건이 노이즈.

## 문제 상황

### 중복 패턴 분석

| 패턴 | 원본 건수 | 실제 가치 | 원인 |
|------|----------|-----------|------|
| strncpy 널 종료 | 9건 | 대표 1건 | 5개 함수가 각각 보고 |
| 스레드 안전성 | 2건 | 0건 | Proframe 단일스레드 |
| NULL 체크 | 2건 | 대표 1건 | 2개 함수가 각각 보고 |
| svc_cnt 경계 | 2건 | 대표 1건 | 미초기화 + 경계 미검사 동일 근본 원인 |
| DB 배열 경계 | 2건 | 대표 1건 | 2개 루프가 각각 보고 |
| ix 미초기화 | 1건 | 0건 | for에서 초기화됨 (오탐) |

### 근본 원인
함수별 개별 LLM 호출(T21)에서 각 함수의 LLM이 **다른 함수의 결과를 모르므로** 동일 패턴을 독립적으로 보고.
프롬프트에 "동일 패턴 병합" 지시를 해도 각 LLM은 자신의 함수만 보므로 적용 불가.

## 해결 (T29)

### `_deduplicate_issues()` 후처리

LLM 결과 합산 후, issue_id 재번호 전에 후처리:

1. **스레드 안전성 자동 제거**: title에 "스레드/동기화/mutex/경쟁 상태" → 삭제
2. **키워드 그룹 병합**: strncpy/NULL 체크/ix 미초기화 → 대표 1건 + "(외 N곳)"
3. **동일 변수 병합**: svc_cnt/currsvclist_s/out_04 + 동일 category → 대표 1건
4. **severity 최고 유지**: 그룹 내 가장 높은 severity를 대표 이슈에 적용
5. **severity 순 정렬**: 결과를 critical → high → medium → low 순으로 재정렬

### 효과
- 29건 → ~15건 (14건 노이즈 제거)
- 핵심 버그(svc_cnt 미초기화, 배열 경계, memcpy 크기 불일치 등)에 집중

## 수집 대상

### 1. Import/Include 정보
각 파일에서 외부 모듈/파일을 참조하는 구문을 모두 추출합니다:
- JavaScript: `import ... from '...'`, `require('...')`
- C: `#include "..."`, `#include <...>`
- Pro*C: `EXEC SQL INCLUDE ...`
- SQL: 없음 (참조 관계 없음)

각 참조에 대해:
- `statement`: 원본 구문
- `resolved_path`: 분석 대상 파일 중 매칭되는 경로 (없으면 null)
- `is_external`: 시스템/외부 라이브러리 여부

### 2. 함수 호출 정보
파일 내에서 호출하는 주요 함수를 추출합니다:
- 함수명, 호출 라인 번호
- 호출 대상이 분석 대상 파일에 정의된 경우 `target_file` 설정

### 3. 코드 패턴 감지
아래 패턴 유형을 감지합니다:
- `error_handling`: try-catch, if-return 에러 처리, WHENEVER SQLERROR
- `logging`: console.log, printf, syslog 등 로깅 구문
- `transaction`: BEGIN/COMMIT/ROLLBACK, EXEC SQL COMMIT
- `memory_management`: malloc/free, new/delete, calloc/realloc

## 주의사항
- 선택된 파일만 분석합니다. 선택되지 않은 파일의 내용을 추측하지 마세요.
- `resolved_path`는 분석 대상 파일 목록 내에서만 매칭합니다.
- `common_patterns`는 전체 파일에 걸친 패턴 유형별 빈도를 집계합니다.
- `dependencies`는 실행 계획에서 전달받은 의존성 그래프를 그대로 포함합니다.
- 이슈가 없더라도 빈 배열 `[]`로 응답하세요 (null 금지).
